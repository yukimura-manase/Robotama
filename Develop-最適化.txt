1. ショートカットまとめ🔥

[ Editor系統でのショートカット🔥 ]

コピー: command C

Windows => Ctrl C

ペースト・貼り付け: command V

切り取り: command X

前に戻る: command X

全選択: command A

検索: command F

Windows => Ctrl F

保存: command S


インテンド: Tab

インテンドをバックする(戻す): Shift Tab

2. 開発効率を上げるためのポイント🔥


[ 意識面・行動面 ]


1. 構造を分析する癖を持つ: 構造を分析するようにする🔥

    => どういう関係性があるのか？

    => どういう構成要素を持っているのか？

    => 紙やEditorに書き出す🔥


2. 未来に向けた計画性: 目的・目標設定および、中間地点などのルーティング・設定をする🔥

    => 未来に向けた計画性🔥

    => 目的・目標の明文化・言語化
    
    => そこに向かうまでの道のり(過程・プロセス)をしっかりとルーティングする・設定する🔥


3. Task管理をする: Editor(VSCode) や Note で、Task管理をする🔥

    => Task-Management.txt => タスクを管理して、クリアしていく楽しさを育てる🔥

    => 明文化・文章化することで、考えが広がったり、見えてくることがある！

    => 文章化することで、凡人でも考えることができる。

    => 頭の中で完結するのは、一部の天才だけです！


4. 小さい単位で試す: 小さい単位で、Testをしながら前に進む🔥

    一気に実行しようとしてもエラーになることが多いです。

    Goal(目標)に向けて、Step By Step で進んでいきましょう🔥



5. Skillの熟成: 学んだことをOutputする 

    => 1. Qiita, Zenn, Blog などでMyDocumentを作成🔥

    => 2. 個人開発などでOutputする🔥

    => Skillが磨かれる🔥



6. コスパのいい！楽しむメンタル: 学ぶことやエンジニアリングを楽しむ🔥

    => Game感覚で楽しむ🔥

    => RPG感覚で、スキルアップ = レベルアップ

    => 真面目に真剣に楽しむ🔥

    => 楽しむことの方が成長よりも大事だし、結果として効果を発揮する🔥

    => メンタルの健康面や、持続性もある🔥



7. 報連相 のタイミング & 適切に頼る力 => コミュニケーションは、最適にする🔥

    7-1. 報連相を徹底する🔥

        => 報連相は、コミュニケーションである！

        => 進捗報告はこまめに！
        
        => Clear状況や取り組んでいることや、今後のTaskなどを雑談ぐらいの感じで話したりでいい。

        報告は、いいこともよくないことも両面、提示する！

        連絡は、関係する人に、情報共有としてしっかりする！

            => あとから参照しやすいように、複雑なことなどはDocumentを作ってもいいと思います。

        相談は、時間を決めて自己解決できない問題に関してする！ =>  適切に頼る力🔥



    7-2. 適切に頼る力(相談力)🔥

        => エラーなど、30分以上悩むような問題は、すぐに報告する🔥

        => 場合によっては、適切に助けを求める！

        => 一緒にエラー解決に動いてもらったり、できる人・わかる人に観てもらったりする🔥
