API エンドポイント（URI） の設計
Web API におけるエンドポイントは、API に アクセスするための URI を指します。

基本的には、URI が「リソース」を指すものであり、URI と HTTP メソッドの組み合わせで処理の内容を表すのが良い設計であるとされています。
これは、HTTP の URI がリソースを指すことで「名詞」として考え、それに HTTP メソッドを「動詞」として組み合わせることで、シンプルに行いたいことを表現することができるからです。

< Laravel-API基本情報 >

0.API名

    Exment-Laravel-API

2. 機能概要（API機能）

    PostgreSQLおよびExment-MySQLと接続しているAPI

3. ベースURI

    未設定

4. アクセス方法

    HTTP通信

5. Exment-Laravel-APIによるデータの流れ(データ・フロー)

    0. 管理者アプリ(Exment)からPostgreSQLのデータを同期するフロー
    
        Exment => Laravel-API => PostgreSQL => Laravel-API => MySQL => Exment

    1. Exmentのパラメータ編集機能によるフロー
    
        Exment => MySQL => Exment

        Exment => Laravel-API => PostgreSQL => 他のアプリ 

    2. 他のアプリからMySQLのデータをread, writeするフロー(和暦検索など)
    
        他のアプリ => Laravel-API => MySQL => Laravel-API => 他のアプリ
