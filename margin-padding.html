1. CSSのmarginとpaddingとは？

CSSでは要素のまわりの余白をmargin（マージン）とpadding（パディング）というプロパティで指定します。

1-1.marginとpaddingの違いは？

padding：要素の内側の余白

=> padding : 詰め物, 綿

margin：要素の外側の余白

=> margin : 余白・余地

1-2. CSSの余白の考え方（ボックスモデル）

要素の中身（内容）があり、そのまわりの余白がpaddingです。
そのまわりにはborder（線）があり、 それより外側の余白がmarginです。

要素 => padding => border(境界線) => margin

「オレンジ色の部分」と「線の部分」が合わせて要素全体になります。
その外側のmarginは要素外の余白なわけですね。
この図のようなCSSの余白の考え方は「ボックスモデル」と呼ばれます。

要素に対してwidthとheightの値を指定した場合には、要素の中身の部分の大きさが変わります。

3. marginとpaddingの値を %で指定する

3-1. % 指定の考え方と計算

% = 親要素の幅に対する比率🔥

%は親要素の幅に対する比率で計算されます。
例えば親要素の幅（width）が1000pxの場合、40% ＝ 400pxになります。

親要素の幅が変われば%指定されているpaddingやmarginの幅も変わります🔥

レスポンシブ対応に最適🔥

%指定するときには、横の%の合計が100%になるようにします（でないとはみ出てしまいます）。「margin30%、padding30%、width40%」という具合ですね。

そのため「paddingを%指定したい」というときは、基本的にはmarginもwidhtも合わせて%指定する必要があります（でないと合計100%になりません）。

%指定するロボ玉🔥

divで親要素を作り、その中に段落要素（p）を入れています。
この段落要素のmarginを左右20%ずつ、widthを60%（計100%）に指定してみます。

3-2. %指定の使いどころは？

ページの内容全体を表すbody要素の幅は、pxで指定されていないとブラウザの幅になります。
つまり、幅1200pxのブラウザでウェブページを見ればbodyの幅は1200pxに、幅400pxのスマホブラウザでページを見ればbodyの幅は400pxになるわけですね。
そのため、ウェブページを開いたブラウザの幅に対して％指定をすることができるのです。

4. marginの相殺

4-1. margin指定が複数並ぶと１つしか適用されない

marginが衝突したときは、値が大きい方が適用される🔥

4-2. 親要素ともmarginの相殺は起こる

「marginが並ぶとそのうちの１つしか適用されない」、また「要素間の余白が最も大きくなるmarginが適用される」のだと覚えておきましょう。

4-3. paddingでは相殺は起こらない
